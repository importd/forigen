# Forigen — 西方马克思思想谱系 3D 地球

## 技术栈

| 层 | 技术 |
|---|---|
| 框架 | React 18 + TypeScript |
| 构建 | Vite 8 |
| 3D 引擎 | Three.js + react-three-fiber + @react-three/drei |
| 数据 | 本地 MD 文件 (YAML frontmatter) + localStorage |
| 部署 | GitHub Pages + GitHub Actions |

## 核心交互

| 功能 | 实现 |
|---|---|
| 3D 地球 | Three.js 球体 + NASA 纹理 + 国界线 + 大气光晕 |
| 思想家节点 | 经纬度定位，学派色圆点，缩放自适应大小 |
| 标签显示 | 缩放到一定距离自动显示姓名，悬停显示 |
| 影响关系线 | 实线 + 滑动亮段表示流向，缩放淡出 |
| 时间轴 | 拖动/播放过滤显示对应年份的思想家 |
| 思想家卡片 | 模块化：通用模块 + 学派分支特有模块 |
| 笔记 | Markdown 编辑 + 实时预览，自动保存 localStorage |
| 搜索 | 中英文 + 核心思想关键词搜索 |
| 数据导入/导出 | MD 文件 zip 打包，含所有元数据和笔记 |

## 数据流

```
seed thinkers (thinkers.ts)  +  customThinkers (localStorage)
         ↓
    useData hook (年份过滤 + 关系图构建)
         ↓
   ┌─────┴─────┐
   ↓           ↓
GlobeScene   DetailPanel
(3D 渲染)    (卡片 UI)
```

笔记和自定义数据存储在 `localStorage`，键名：
- `forigen-notes` — 笔记
- `forigen-custom-thinkers` — 自定义思想家
- `forigen-custom-labels` — 自定义标签（学派/地区/思想）
- `forigen-custom-idea-details` — 自定义学术阐释
- `forigen-custom-school-theories` — 自定义学派理论

## 部署方案

### GitHub Pages + Actions

**配置：**

`vite.config.ts`:
```ts
export default defineConfig({
  base: '/forigen/',  // GitHub Pages 项目路径
  plugins: [react()],
})
```

`.github/workflows/deploy.yml`:
```yaml
name: Deploy to GitHub Pages
on:
  push:
    branches: [master]
jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20 }
      - run: npm ci
      - run: npm run build
      - uses: actions/configure-pages@v4
      - uses: actions/upload-pages-artifact@v3
        with: { path: dist }
      - uses: actions/deploy-pages@v4
```

**部署地址：** `https://importd.github.io/forigen/`

**更新流程：** `git push origin master` → Actions 自动构建部署（约 30 秒）

### 本地开发

```bash
npm install        # 安装依赖
npm run dev        # 启动开发服务器 (localhost:5173)
npm run build      # 生产构建 → dist/
```

### 资产路径注意

所有硬编码的静态资源路径（`useTexture`、`fetch` 等）必须使用 `import.meta.env.BASE_URL` 前缀，否则部署到子路径（如 `/forigen/`）时会 404：

```ts
// ✅ 正确
useTexture(import.meta.env.BASE_URL + 'textures/earth.jpg')
fetch(import.meta.env.BASE_URL + 'data/countries.geojson')

// ❌ 错误（部署到子路径时找不到）
useTexture('/textures/earth.jpg')
```
